Functions

Topic id gmat::quant::algebra::functions · Domain Algebra · Prereqs algebraic expressions, linear equations · ~22 min

Learning objectives

  • Read and use function notation f(x); evaluate by substitution.
  • Evaluate compositions f(g(x)) from the inside out.
  • Interpret GMAT-style custom operation symbols.
  • Solve for an input given an output, and find an unknown inner function.

Model: I do → we do → you do · application-first (explanations revealed after an attempt) · Rosenshine (daily review), Archer (I-do/we-do/you-do), SPOV2 application-first.

1Opening — retrieval & bridge

Builds on: Evaluating expressions by substitution.

Do Now · from memory, no notes

From memory, no calculator. 4 minutes.

  1. If f(x) = 2x + 1, find f(3).7.
  2. If g(x) = x^2, find g(−4).16.
  3. Evaluate 3a − 2 at a = 5.13.
  4. If f(x) = x − 4, find f(4).0.
  5. In f(g(2)), what do you compute first?g(2) first (inside out), then apply f.

Retrieval starter (5 min). Retrieves substitution (the only mechanical skill functions need) and the inside-out order that composition and custom symbols depend on.
If students struggle: If notation feels alien, read f(3) as 'the rule f applied to 3' — substitute 3 wherever x appears.

Prior-knowledge bridge

A function is just a substitution rule with a name. You've evaluated expressions for years; f(x) only labels the rule so we can chain rules. Q5 sets up the one new habit: in f(g(x)) you work the inside first. Custom GMAT symbols (like a ◇ b) are the same idea wearing a costume.

Learning intention

By the end you can evaluate functions and compositions, decode custom operation symbols, and solve for an input or an inner rule.

You'll know you've got it when:

  • I can evaluate f(x) for any input, including negatives.
  • I can compute f(g(x)) inside out.
  • I can apply a defined custom operation correctly.

2I do — worked example

Watch the reasoning modeled end to end before you try.

I DO · worked examplemedium
If f(x) = 2x + 3 and g(x) = x^2, what is f(g(2))?
  • A7
  • B11
  • C14
  • D19
  • E49

Think-aloud

  1. Composition works from the inside out — do g(2) first.
  2. g(2) = 2^2 = 4.
  3. Now apply f to that result: f(4) = 2·4 + 3 = 11.

Correct: B. Inside out: g(2) = 2^2 = 4, then f(4) = 2(4) + 3 = 11.

Takeaway. For f(g(x)), evaluate the inner function first, then substitute that value into the outer function.

3We do — guided practice

Scaffolds and immediate feedback are one click away. Fade them as you gain confidence.

we_1 · we do (guided)easy
If f(x) = 3x − 5, find f(−2).
  • A−11
  • B−1
  • C1
  • D11
  • E−6
Scaffold hints (peek only if stuck)
  1. Substitute x = −2 everywhere.
  2. 3(−2) = −6.
  3. −6 − 5 = −11.
Immediate feedback & explanation

If correct: Right — −11.
If incorrect: Keep the negative: 3 times −2 is −6, then subtract 5.

Answer: A. f(−2) = 3(−2) − 5 = −6 − 5 = −11.

we_2 · we do (guided)medium
A custom operation is defined by a ◇ b = a^2 − b. What is 5 ◇ 3?
  • A22
  • B13
  • C28
  • D2
  • E8
Scaffold hints (peek only if stuck)
  1. Match the symbols: a = 5, b = 3.
  2. a^2 = 25.
  3. 25 − 3 = 22.
Immediate feedback & explanation

If correct: Yes — 22.
If incorrect: Square the first input (a), then subtract the second (b).

Answer: A. Replace a with 5 and b with 3: 5^2 − 3 = 25 − 3 = 22.

5Mastery check

A topic is mastered at >=85% correct on new/held-out-style questions across >=2 spaced sessions with delay (PRD Section 9.2). Flashcard recall alone is not mastery; application under mixed, delayed conditions is the bar.

Threshold 0.85 · spaced sessions ≥ 2

  • >=85% correct on new/held-out-style items (not flashcard recall) per session
  • sustained across >=2 spaced sessions with a delay between them
  • per-topic memory surfaced only after >=20 graded reviews (PRD give-up rule)
  • per-topic performance surfaced only after >=8 application attempts (PRD give-up rule)

If not yet mastered: Not yet: route every miss to the required error log (tag why: content gap / reasoning flaw / timing / careless / trap), schedule spaced re-practice via topic-aware scheduling, then re-enter the we-do -> you-do loop before re-testing.
